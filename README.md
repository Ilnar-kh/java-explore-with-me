Explore With Me

О проекте

Explore With Me — это backend-приложение для публикации и поиска событий.
Пользователи могут создавать мероприятия, находить интересные события, оставлять комментарии и собирать компанию для участия.

Проект реализован в формате микросервисной архитектуры и разработан в рамках дипломного проекта (Java / Spring).

⸻

Архитектура

Проект состоит из двух микросервисов, каждый с собственной базой данных:

1. Основной сервис (порт 8080)

Отвечает за бизнес-логику приложения:
	•	управление событиями (создание, редактирование, публикация);
	•	категории и подборки событий;
	•	пользователи и их заявки на участие;
	•	комментарии к событиям (дополнительная функциональность);
	•	разграничение доступа: публичный / пользовательский / административный API.

2. Сервис статистики (порт 9090)
	•	фиксирует обращения к публичным эндпоинтам;
	•	хранит статистику просмотров событий;
	•	предоставляет агрегированную аналитику.

⸻

Реализованная функциональность

Публичный API
	•	просмотр списка событий;
	•	фильтрация и сортировка;
	•	просмотр подробной информации о событии.

API авторизованных пользователей
	•	создание и управление событиями;
	•	подача заявок на участие;
	•	управление своими комментариями.

Административный API
	•	модерация событий;
	•	управление пользователями;
	•	управление категориями и подборками;
	•	модерация комментариев.

⸻

Дополнительная фича: комментарии

В рамках расширения проекта реализована отдельная функциональность комментариев к событиям:
	•	пользователи могут оставлять комментарии;
	•	редактировать и удалять свои комментарии;
	•	администратор может модерировать комментарии;
	•	реализована в отдельном пакете с контроллерами, сервисами, DTO и репозиториями;
	•	покрыта Postman-тестами.

⸻

Запуск проекта локально
	1.	Склонировать репозиторий
	2.	Собрать проект:

mvn clean package


	3.	Запустить контейнеры:

docker-compose up -d


	4.	Основной сервис будет доступен на http://localhost:8080
Сервис статистики — на http://localhost:9090


Назначение проекта

Проект демонстрирует:
	•	работу с микросервисной архитектурой;
	•	проектирование REST API;
	•	разделение ответственности между сервисами;
	•	работу с БД и валидацией данных;
	•	написание расширяемой бизнес-логики.
